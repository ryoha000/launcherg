# Launcherg Server Stack

## Summary

`F:\workspace\launcherg\server` is a Cloudflare-first TypeScript service.
It serves both an API and a small web UI from the same deployment.

## Runtime and Build

- Cloudflare Workers runtime via `wrangler`
- TypeScript with ESM
- Vite for the UI build
- Vitest for tests

## API and Validation

- Manual request routing in `src/app.ts`
- Shared contract and validation types in `src/shared/schema.ts`
- oRPC contract definitions in `src/shared/router.ts`
- Zod schemas for request and response validation

## UI

- React 19
- React DOM 19
- Vite React plugin
- SPA assets built from `ui/` and served through Workers assets

## Storage and State

- D1 for devices, sync snapshots, and deduplicated image metadata
- R2 for image binaries
- Signed cookie session flow instead of a database-backed session store

## Existing Free-Tier-Friendly Techniques

- API and SPA are consolidated into one Worker deployment
- Static UI is served from assets instead of hand-written HTML responses
- Binary images are stored in R2, not D1
- Uploads use presigned R2 `PUT` URLs generated by the Worker
- Image uploads are deduplicated by stable keys before storage
- Snapshot writes skip unchanged rows
- D1 lookups for large key sets are chunked

## Guidance For This Repository

- Preserve the current split: metadata in D1, blobs in R2, UI in assets.
- Prefer extending the current Worker over adding another Cloudflare service.
- Keep sync incremental and deduplicated.
- Treat new recurring jobs, queues, and durable coordination as exceptions that require explicit justification.
